# 企業評価プラットフォーム「-0（ゼロ）」計画書

就職者のための**適正な企業評価**と**ブラック企業の可視化**を行う Web アプリケーション。

## 1. コンセプト

求人広告やイメージだけでは分からない「実態」を、複数の労働指標から**透明なスコア**で可視化する。
特に**ブラック企業リスク**を定量化し、利用者ができるだけ危険な企業に入らないよう警告する。

- 就職者側: 企業を客観指標で比較し、ミスマッチ・ブラック企業を回避
- 評価の公正さ: スコアの根拠（各指標の寄与）を必ず開示（ブラックボックスにしない）

## 2. スコープ（MVP）

| 機能 | 内容 |
| --- | --- |
| 企業検索 | 名称・業界・地域でのフィルタ／キーワード検索 |
| ブラック度スコア | 労働指標から 0–100 で算出し、4段階のリスク区分で表示 |
| 適正評価 | 良い点・懸念点・総合所見を透明に提示 |
| 企業詳細 | 各指標の実数値、レーダー、赤信号（レッドフラグ）一覧 |
| 企業比較 | 複数社を並べて比較 |
| お気に入り | localStorage に保存（サーバー不要） |
| ソート | ブラック度／ホワイト度／残業／離職率などで並び替え |

## 3. 技術構成

- フロント: Vite + React + TypeScript
- スタイル: 自前CSSデザインシステム（依存を最小化しビルドを安定）
- テスト: Vitest（スコアリングエンジンのユニットテスト）
- データ: リアルな架空企業のシードデータ（`src/data/companies.ts`）
  - データ層を分離し、将来は API / DB / ユーザー投稿へ差し替え可能

## 4. ブラック企業スコアリング（中核ロジック）

各指標を「リスクポイント 0–100」に正規化し、重み付き平均で**ブラック度スコア**を算出。
ホワイト度 = 100 − ブラック度。

| 指標 | 重み | 根拠 |
| --- | --- | --- |
| 月平均残業時間 | 0.20 | 45h超で警告、80h超で過労死ライン |
| 3年以内離職率 | 0.18 | 高いほど定着せず＝実態が悪い |
| 有給休暇消化率 | 0.12 | 低いほど休めない |
| 残業代支給率 | 0.12 | 低い＝サービス残業の疑い |
| 平均勤続年数 | 0.10 | 短い＝使い捨て傾向 |
| ハラスメント指標 | 0.10 | 報告件数（従業員比） |
| 労基署是正勧告 | 0.10 | 直近5年の法令違反歴 |
| 万年採用 | 0.04 | 常時大量採用は高離職の兆候 |
| 社保完備 | 0.04 | 未整備は基礎的な問題 |

### リスク区分

- 0–24: 優良（ホワイト）
- 25–44: 標準
- 45–64: 要注意
- 65–100: ブラック危険

### レッドフラグ（即警告する明示条件）

- 月残業 80h 超（過労死ライン）
- 残業代支給率が低くサービス残業の疑い
- 3年離職率 30% 超
- 労基署の是正勧告あり
- 社会保険 未整備
- 万年採用 かつ 高離職率

## 5. 実データ化（Wikidata）

労働環境（ブラック度）と将来性の双方を、可能な範囲で**実データ**に接続する。

- **データ源**: Wikidata SPARQL（無認証・CC0）。日本の実在企業の事実データ
  （従業員数・設立年・売上高の推移・業種・本社・上場・公式サイト）を取得。
- **取得パイプライン**: `scripts/fetch-companies.mjs` が SPARQL を実行し、
  名寄せ・業種正規化・所在地整形を行い `src/data/companies.generated.json` を生成。
  従業員数・売上高は複数年の値を保持し、成長率（CAGR）算出に用いる。
- **データ層の分離**: `src/data/index.ts` が「実データ（Wikidata）」と
  「デモ（労働環境つき架空データ）」の 2 系統を提供。UI から切替可能。

### 労働データの扱い（重要な設計判断）

Wikidata には残業・離職率などの労働指標が無い。**実名企業に推測値を付与すると
名誉毀損リスクがある**ため、実データ企業には労働指標を付与せず「未連携」とする。
ブラック度は、公的労働データ（厚労省しょくばらぼ / gBizINFO 等、無料 API トークンが必要）
の連携時のみ算出する方針。デモデータではフルにブラック度評価を体験できる。

## 6. 将来性（成長性）分析

`src/engine/growth.ts`。実データから「将来性スコア 0–100（高いほど有望）」を算出。
データ欠損の要因は**重みを動的に再正規化**して除外し、各要因の寄与を開示する。

| 要因 | 基準重み | 入力 |
| --- | --- | --- |
| 業種の将来性 | 0.30 | 業種アウトルック表（`industry.ts`、編集可能） |
| 売上成長率 | 0.24 | 売上高 CAGR（外れ値除外） |
| 従業員数の成長 | 0.14 | 従業員数 CAGR |
| 成長ステージ | 0.14 | 企業年齢（設立年） |
| 事業規模の安定性 | 0.08 | 従業員数 |
| 資本アクセス | 0.10 | 上場の有無 |

成長ステージ: 急成長期 / 成長期 / 成熟・安定 / 転換期・要注意。

## 7. 多面評価（生産性・株/投資・働きやすさ）

将来性・ブラック度に加え、複数の観点で企業を評価する（データが揃う軸のみ表示）。

- **生産性** `src/engine/productivity.ts` — 一人当たり売上高（実データ）を対数スケールで
  0–100 に正規化。営業利益・営業利益率も算出。
- **株・投資** `src/engine/stock.ts` — 売上・純利益・純利益率・時価総額・上場市場の
  財務スナップショット。取得できた範囲のみ表示し、詳細は EDINET / J-Quants 連携で補完。
  投資助言ではない旨を明示。
- **働きやすさ** `src/engine/workability.ts` — 残業の少なさ・有給・定着・勤続・多様性から
  ワークライフ重視のスコア（労働データがある企業のみ）。

## 8. 実労働データ連携

**働きやすさ（残業・有給・女性管理職）はトークン不要の公開データで連携済み。**
`scripts/enrich-positive-db.mjs` が厚労省 女性活躍・両立支援DB のオープンデータCSVを
法人番号（Wikidata P3225）で突合し、実名企業に `laborReal` を追記する。

ブラック度（離職率・残業代・法令違反）や平均年収は `scripts/enrich-labor.mjs` +
`.env.example` を用意（しょくばらぼCSVは無認証、gBizINFO / EDINET は無料トークン）。
未設定なら安全に no-op。詳細は [`src/data/sources/README.md`](../src/data/sources/README.md)。

## 8b. データ分析ダッシュボード

`scripts/enrich-shokuba.mjs` がしょくばらぼ全データ（約15万社・Shift-JIS・634列）を
ストリーム解析して `analytics.generated.json`（業種別・都道府県別平均、残業分布）を生成。
アプリの「データ分析」ビュー（`src/components/Dashboard.tsx` + `charts.tsx`）で
全国の実データと掲載企業群を可視化する（棒・ヒストグラム・散布図・ランキング）。

## 9. 見やすさ（人間中心のUI）

- グレード表示（S/A/B/C/D）で一目で評価がわかる。すべて「高い＝良い」向きに統一。
- 詳細画面はタブ構成（総合／将来性／生産性・財務／働きやすさ／リスク）。総合タブに
  平易な一言コメントと強み・気になる点の要約を表示（`src/engine/overview.ts`）。
- 「スコアの見方」凡例を常設。
